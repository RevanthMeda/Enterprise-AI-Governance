## Next 3 to build

### 1. Search & Filtering Across All Pages

This is the highest-value next step.

Why now:

* Your surface area has expanded a lot: registry, detail pages, controls, workflows, audit logs, exports.
* Without strong search/filtering, the product starts feeling heavy as real customer data grows.
* This improves daily operator efficiency immediately.

What to add:

* global search by system name, owner, vendor, department
* filters for risk level, status, framework, geography, data sensitivity
* audit log date range + actor + action type
* saved filter presets
* URL-persisted filters for shareable views

Best implementation detail:

* keep filtering server-side for correctness and scale
* mirror filter state in query params
* add indexed columns in Postgres for common filter fields

### 2. Notification System

This becomes much more valuable now that auth and roles exist.

Why now:

* you now know who should receive what
* workflows and overdue controls can actually trigger useful alerts
* it increases product stickiness and operational follow-through

Start with:

* pending approval assigned to reviewer
* control due soon / overdue
* workflow status changed
* evidence requested
* high-risk system created or modified

Recommended approach:

* in-app notifications first
* email second
* event-driven notification service interface so Slack/Teams can be added later

### 3. Evidence File Uploads

This is the next biggest enterprise credibility upgrade.

Why now:

* you already have exports, audit history, and system detail pages
* evidence attachments complete the audit story
* regulators, internal audit, and procurement teams love proof artifacts

Support first:

* upload evidence against control
* upload evidence against workflow/review
* file metadata: uploader, timestamp, linked system, linked control
* version history / replacement behavior
* downloadable evidence list in PDF report

Implementation note:

* abstract file storage immediately
* local disk for dev, S3-compatible interface for later
* do not hardcode filesystem assumptions into business logic
